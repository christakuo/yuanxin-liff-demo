# Task 03A-10：Google Sheet 實際欄位新增操作方案

## 一、任務目的

本文件將 Task 03A-9 的欄位調整清單，轉成未來可實際操作的 Google Sheet 欄位新增方案。

本階段目的：

- 讓非工程使用者可依步驟手動新增欄位
- 明確說明新增順序、欄位位置、備份方式與驗收方式
- 避免正式修改 Google Sheet 時誤刪、誤改或覆蓋既有資料
- 作為 Task 03A-11 正式操作前的依據

本階段只建立操作方案，不直接修改 Google Sheet、不修改 Apps Script、不修改前端程式。

## 二、操作前必要條件

正式操作前，必須先確認：

- GitHub `main` 已同步且 `0 changed files`
- Google Sheet 已另存備份
- Apps Script 暫不修改
- 前端暫不修改
- 一次只改一個分頁
- 第一階段只處理 `leads_潛在會員名單`
- 修改前先截圖第一列欄位
- 修改前確認可回復方式
- 修改過程不要同時開多個編輯視窗
- 修改後先做小量測試，不直接批次轉換全部資料

## 三、Google Sheet 備份方式

正式修改前，請先備份 Google Sheet。

操作方式：

1. 打開正式 Google Sheet：

```text
元馨醫管家_平台主資料庫_v2
```

2. 點上方選單：

```text
檔案 > 建立副本
```

3. 副本命名建議：

```text
元馨醫管家_平台主資料庫_v2_備份_YYYYMMDD_Task03A
```

例如：

```text
元馨醫管家_平台主資料庫_v2_備份_20260617_Task03A
```

4. 確認備份檔可以正常開啟。
5. 備份檔只作為回復用途，不在備份檔上正式作業。
6. 正式修改仍在原正式檔進行。

## 四、leads_潛在會員名單 欄位新增原則

第一階段只做：

```text
新增欄位
```

不做：

```text
刪除欄位
改欄位名稱
搬移既有欄位
覆蓋既有資料
批次轉換全部資料
```

新增原則：

- 新欄位建議加在現有欄位最右側
- 避免插入中間造成 Apps Script 欄位位置誤判
- 既有欄位 `名單狀態` 保留
- 新增 `CRM階段` 作為正式 11 階段欄位
- 先讓舊欄位與新欄位並行
- 等 Apps Script 與前端確認穩定後，再評估是否停用舊欄位

## 七、CRM階段 下拉選單設定

`CRM階段` 欄位正式下拉選單建議設定為：

```text
01 新名單
02 待首次聯繫
03 已聯繫
04 培育中
05 已預約諮詢
06 方案評估中
07 已簽約（未付款）
08 已成交（付款完成）
09 暫緩
10 無意願
11 停止聯繫／退訂
```

設定原則：

- 下拉選單先只設定在 `CRM階段` 欄位
- 不修改舊 `名單狀態` 下拉選單
- 先測試少量資料
- 不要立刻批次套用舊資料
- 高風險狀態需人工確認

| 是否重複資料 | 否 | 重複時再改是 |

## 九、操作步驟

### 1. 打開正式 Google Sheet

打開：

```text
元馨醫管家_平台主資料庫_v2
```

### 2. 建立備份

依第三章方式建立副本，確認備份可開啟。

### 3. 切到 leads_潛在會員名單

在底部工作表分頁，點選：

```text
leads_潛在會員名單
```

### 4. 確認第一列欄位

確認第一列是欄位名稱，並截圖保存。

### 5. 到最右側新增欄位

移到目前資料表最右側，在最後一欄後面新增欄位。

注意：

```text
不要插入中間
不要改舊欄位名稱
不要刪除舊欄位
```

### 6. 先新增第一批欄位

依第五章表格新增第一批欄位。

新增後檢查欄位名稱是否完全一致。

### 7. 設定 CRM階段 下拉選單

只針對 `CRM階段` 欄位設定 11 階段下拉選單。

不要修改舊 `名單狀態`。

### 8. 設定建議預設值

先在少量測試列填入建議預設值，不要直接批次填滿全部資料。

### 9. 小量測試 3 到 5 筆資料

建議選 3 到 5 筆不同狀態資料測試：

- 新名單
- 已聯繫
- 已預約諮詢
- 已簽約
- 已成交

確認 CRM階段 能正確選擇。

### 10. 檢查顧問入口與推薦事件

修改欄位後，需確認：

- 顧問入口可正常開啟
- 顧問業務大廳可正常顯示
- 推薦事件仍可寫入
- 會員首頁不受影響

- [ ] 欄位都新增在最右側

- [ ] `CRM階段` 下拉選單正確
- [ ] 沒有資料被覆蓋
- [ ] Apps Script 尚未修改
- [ ] 前端尚未修改
- [ ] 顧問入口可正常開啟
- [ ] 顧問業務大廳可正常顯示
- [ ] 推薦事件仍可寫入
- [ ] 會員首頁仍可正常開啟
- [ ] 可回復備份
- [ ] 已截圖保留修改後狀態

## 十一、高風險提醒

正式操作時務必注意：

- 不要刪除欄位
- 不要改欄位名稱
- 不要把 `CRM階段` 插到表格中間
- 不要直接覆蓋 `名單狀態`
- 不要在未備份前修改
- 不要一次修改多個分頁
- 不要立即批次轉換全部資料
- 不要把 `已簽約` 直接改成 `已成交`
- 不要把 `放棄追蹤` 全部改成 `停止聯繫／退訂`
- 不要讓停止聯繫客戶繼續收到行銷訊息
