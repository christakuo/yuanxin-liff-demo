# Task 02C｜顧問資源中心／業務大廳完整版規劃

## 一、Task 02C 目標

Task 02C 的目標，是在不影響 Task 02B 顧問業務大廳 MVP 的前提下，逐步將目前雲端共享資料夾中的顧問資源整合至 LIFF 顧問業務大廳。

未來健康顧問可在同一個安全入口中查看：

- 個人業務數據
- 自己歸屬的推薦名單
- 自己推薦的顧問
- 團隊資訊與發展進度
- 招募計畫與說明文件
- 獎金制度
- 顧問培訓素材
- 行銷素材
- 健康專欄素材
- 常用話術
- 重要公告

本階段以「資訊架構、資料結構及後續開發順序」為主，不直接替換既有入口，也不進行大幅前端改版。

## 二、商業目標

1. 降低顧問尋找文件與素材的時間。
2. 建立統一、可控管、可更新的顧問資源入口。
3. 讓顧問能快速掌握自己的名單、團隊與業務進度。
4. 降低營運團隊重複傳送文件及回覆相同問題的工作量。
5. 為未來顧問分級權限、公告推播、培訓追蹤及業績管理建立基礎。

## 五、建議新增 Google Sheet 分頁

### 分頁名稱

`consultant_resources_顧問資源中心`

### 建議欄位

| 欄位 | 用途 | 範例 |
|---|---|---|
| `resourceId` | 資源唯一編號 | `RS000001` |
| `category` | 資源分類 | `培訓中心` |
| `title` | 顯示標題 | `新人入門訓練手冊` |
| `description` | 簡短說明 | `新進顧問必讀的服務流程與平台操作說明` |
| `fileUrl` | 檔案或網頁連結 | Google Drive、YouTube 或網站 URL |
| `type` | 資源類型 | `文件`、`圖片`、`影片`、`連結` |
| `visible` | 是否在前台顯示 | `是`／`否` |
| `sortOrder` | 同分類排序 | `10` |
| `updatedAt` | 最後更新時間 | `2026-06-14 10:00:00` |

### 資料管理原則

1. 前台只讀取 `visible = 是` 的資料。
2. 先依分類排序，再依 `sortOrder` 由小到大排列。
3. `resourceId` 建議採用 `RS` 加六位流水號。
4. 雲端共享資料夾在過渡期間繼續保留，`fileUrl` 可先連回現有檔案。
5. 文件更新時，以更新 Sheet 資料為主，不必每次修改前端程式。
6. 未來若有權限差異，可再增加 `visibleRanks` 或 `minimumRank`，本階段暫不加入。

## 六、建議新增 Apps Script API

### API 名稱

`getConsultantResources`

本階段只規劃，不實作。

### 建議職責

1. 驗證顧問 LINE 身分。
2. 確認顧問狀態為可使用業務大廳的有效狀態。
3. 讀取 `consultant_resources_顧問資源中心`。
4. 只回傳 `visible = 是` 的資源。
5. 依 `category` 及 `sortOrder` 排序。
6. 使用固定白名單回傳欄位，避免暴露內部資料。

### 建議請求

```json
{
  "action": "get_consultant_resources",
  "lineAccessToken": "單次使用的 LINE access token"
}
```

### 建議回傳

```json
{
  "status": "success",
  "resources": [
    {
      "resourceId": "RS000001",
      "category": "培訓中心",
      "title": "新人入門訓練手冊",
      "description": "新進顧問必讀資料",
      "fileUrl": "https://example.com/resource",
      "type": "文件",
      "sortOrder": 10,
      "updatedAt": "2026-06-14 10:00:00"
    }
  ]
}
```

### 安全原則

- 不採信前台傳入的顧問ID。
- 使用 LINE access token 驗證顧問身分。
- 不儲存或記錄 LINE access token。
- 不回傳 `visible = 否` 的資源。
- 不回傳 Google Sheet 整列原始資料。
- 若驗證失敗，不回傳任何顧問資源。

### 2. 快速操作

- 查看獎金制度

以分類卡片或分頁呈現：

- 招募中心
- 獎金制度
- 培訓中心
- 行銷素材

首頁只顯示近期且重要的公告，未來可增加：

- 公告有效期限

- LINE 推播連動

本階段可先由資源中心的 `重要公告` 分類提供內容。

## 八、建議系統流程

```text
顧問開啟 LIFF 業務大廳
→ 後端驗證 LINE 身分
→ 取得顧問基本資料
→ 載入業務總覽
→ 載入自己的推薦名單
→ 載入顧問資源中心
→ 載入重要公告
→ 前端依模組呈現
```

所有涉及顧問身分及業務資料的查詢，都應由後端以驗證後的 LINE userId 決定資料範圍，不由前台指定顧問ID。
