# Task 02B：顧問業務大廳 MVP 完成紀錄、暫緩項目及後續規劃

## 一、任務背景

「元馨醫管家」目前透過健康顧問招募會員，並希望逐步建立一套高效、自動化、可擴充的顧問業務管理系統。

Task 02A 已完成推薦碼追蹤與 LINE 綁定 Webhook。
Task 02B 的主要目標，是在既有 LIFF 與 Apps Script 架構上，建立「顧問業務大廳 MVP」，讓已開通顧問權限者可以透過 LIFF 查看自己的推薦名單與相關業務資料。

本階段重點是先完成「顧問查詢系統的 MVP 基礎」，不是立即取代現有的雲端共享資料夾。

---

## 二、Task 02B 任務目標

Task 02B 的 MVP 目標如下：

1. 顧問可透過專屬 LIFF 入口進入顧問業務大廳。
2. 系統可依 LINE userId 辨識顧問身份。
3. 已開通顧問權限者可看到自己的推薦名單。
4. 系統可支援後續擴充：

   * 我的推薦顧問
   * 導師查看旗下顧問推薦會員
   * 團隊發展概況
   * 顧問資源中心
   * 獎金與組織儀表板

## 五、暫緩項目

### 1. LINE 官方帳號「啟動業務大廳」關鍵字入口暫不修改

目前官方 LINE 的「啟動業務大廳」仍連到原本的雲端共享資料夾。

暫緩原因：

* 現有共享資料夾內仍有顧問需要使用的資料。
* 內容包含招募資料、獎金制度、培訓素材、健康專欄、行銷工具與其他業務文件。
* 若現在直接改成 LIFF 顧問業務大廳，顧問會看不到原本共享資料夾中的其他資料。
* 因此目前先保留既有流程。

後續處理原則：

```text
等顧問業務大廳完整版完成後，再一次性替換 LINE 官方帳號「啟動業務大廳」入口。
```

未來理想狀態：

```text
啟動業務大廳
↓
進入顧問專屬工作台
↓
同時可查看：
1. 我的推薦名單
2. 我的推薦顧問
3. 團隊會員 / 團隊發展
4. 招募資料
5. 獎金制度
6. 培訓素材
7. 行銷工具
8. 健康專欄
9. 常用話術
10. 重要公告
```

## 六、待辦事項

### 1. 會員端「上傳我的報告」流程修復

在 Task 02A 測試期間曾發現：

```text
使用者在聊天室中點選「上傳我的報告」按鈕後沒有反應。
```

此項目尚未修復，需列為高優先待辦。

可能原因包含：

* LIFF 是否在 LINE App 內正確開啟
* 按鈕事件是否正常觸發
* `liff.sendMessages()` 是否具備正確權限
* LIFF 權限設定是否完整
* 使用者是否已登入 LINE
* 是否需要改成備援流程，例如引導使用者手動回聊天室上傳

建議後續獨立建立任務：

```text
Task 02C-1：會員端健檢報告上傳流程修復
```

修復目標：

```text
會員進入 LIFF
↓
完成身分驗證
↓
點選上傳報告
↓
系統成功帶回 LINE 聊天室
↓
使用者可上傳照片或 PDF 健檢報告
```
